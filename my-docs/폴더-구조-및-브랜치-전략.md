# NFI 다중 거래소 운영 구조

PC에서 여러 거래소를 병렬로 드라이런/라이브 테스트하기 위한 폴더 + Git 브랜치 전략.

## 폴더 구조

```
d:/OneDrive/Project/nostalgia-for-infinity/
├── nfi-binance/   (my-setup-binance 브랜치, 포트 9044)
├── nfi-okx/       (my-setup 또는 my-setup-okx 브랜치, 포트 9046)
└── nfi-bybit/     (my-setup-bybit 브랜치, 포트 9045)
```

각 폴더는 독립된 Git 저장소이며, 각자 Docker 컨테이너로 동시 실행됨.

## Git 브랜치 구조

Fork: `github.com/letjsk/NostalgiaForInfinity`

```
main                    공식(upstream/main) 동기화 전용, 커스텀 없음
my-setup                모든 거래소 공통 커스텀 (페어 수 60, max_trades 8, my-docs/)
├── my-setup-binance    바이낸스 전용 튜닝 (현재는 my-setup과 동일)
├── my-setup-okx        OKX 전용 튜닝 (현재는 my-setup과 동일)
└── my-setup-bybit      Bybit 전용 튜닝 (현재는 my-setup과 동일)
```

### 브랜치별 배치 파일

**my-setup (공통)**:
- `configs/pairlist-volume-*.json` (모든 거래소, 60페어 통일)
- `configs/blacklist-*.json` (모든 거래소)
- `configs/exampleconfig.json` (max_trades 8)
- `my-docs/` (한국어 가이드)
- `.gitignore` 추가 (`.claude/`, `.omc/`)

**my-setup-binance/okx/bybit (거래소별)**:
- 현재는 my-setup과 동일
- 향후 거래소별 특수 튜닝이 필요할 때 여기에 커밋

### Gitignore된 파일 (폴더별 수동 관리)

- `.env` - API 키, 봇 이름, 포트, 텔레그램 토큰
- `user_data/config.json` - pairlist/blacklist 참조 경로
- `user_data/*.sqlite` - 거래 히스토리 DB

## 폴더별 설정 차이

| 항목 | nfi-binance | nfi-okx | nfi-bybit |
|------|-------------|---------|-----------|
| 브랜치 | my-setup-binance | my-setup-okx | my-setup-bybit |
| EXCHANGE__NAME | binance | okx | bybit |
| EXCHANGE__PASSWORD | 없음 | 필수 | 없음 |
| 포트 (LISTEN_PORT) | 9044 | 9046 | 9045 |
| BOT_NAME | NFI_DryRun_Binance | NFI_DryRun_OKX | NFI_DryRun_Bybit |
| pairlist 경로 | pairlist-volume-binance-usdt.json | pairlist-volume-okx-futures.json | pairlist-volume-bybit-usdt.json |
| blacklist 경로 | blacklist-binance.json | blacklist-okx.json | blacklist-bybit.json |
| 텔레그램 봇 | 각자 별도 발급 | 각자 별도 발급 | 각자 별도 발급 |

## 새 폴더 셋업 절차

### 1. 클론

```powershell
cd D:\OneDrive\Project\nostalgia-for-infinity
git clone -b my-setup-<거래소명> git@github.com:letjsk/NostalgiaForInfinity.git nfi-<거래소명>
cd nfi-<거래소명>
```

### 2. `.env` 작성

예시 (nfi-bybit):
```env
FREQTRADE__BOT_NAME=NFI_DryRun_Bybit
FREQTRADE__MAX_OPEN_TRADES=8

FREQTRADE__TRADING_MODE=futures
FREQTRADE__MARGIN_MODE=isolated

FREQTRADE__EXCHANGE__NAME=bybit
FREQTRADE__EXCHANGE__KEY=<BYBIT_API_KEY>
FREQTRADE__EXCHANGE__SECRET=<BYBIT_SECRET>

FREQTRADE__TELEGRAM__ENABLED=true
FREQTRADE__TELEGRAM__TOKEN=<BYBIT_BOT_TOKEN>
FREQTRADE__TELEGRAM__CHAT_ID=<CHAT_ID>

FREQTRADE__API_SERVER__ENABLED=true
FREQTRADE__API_SERVER__LISTEN_PORT=9045
FREQTRADE__API_SERVER__USERNAME=nanjsk
FREQTRADE__API_SERVER__PASSWORD=<PASSWORD>
FREQTRADE__API_SERVER__JWT_SECRET_KEY=<JWT_KEY>
FREQTRADE__API_SERVER__WS_TOKEN=<WS_TOKEN>

TZ=Asia/Seoul
FREQTRADE__DRY_RUN=true
FREQTRADE__STRATEGY=NostalgiaForInfinityX7
```

OKX는 `FREQTRADE__EXCHANGE__PASSWORD=<PASSPHRASE>` 추가 필요.

### 3. `user_data/config.json` 생성

예시 (Bybit):
```json
{
  "strategy": "NostalgiaForInfinityX7",
  "add_config_files": [
    "../configs/trading_mode-futures.json",
    "../configs/pairlist-volume-bybit-usdt.json",
    "../configs/blacklist-bybit.json",
    "../configs/exampleconfig.json",
    "../configs/exampleconfig_secret.json"
  ]
}
```

### 4. 실행

```powershell
docker compose up -d
docker compose logs -f
```

### 5. 접속

`http://localhost:<LISTEN_PORT>`

## 공통 설정 업데이트 흐름

공통 설정 바꿀 일 생기면 (예: 페어 수, max_trades 변경):

```powershell
# 1. 아무 폴더에서 my-setup으로 전환
cd D:\OneDrive\Project\nostalgia-for-infinity\nfi-okx
git checkout my-setup

# 2. 공통 설정 수정 후 커밋
git add configs/exampleconfig.json
git commit -m "config: change max_trades to 10"
git push origin my-setup

# 3. 각 거래소 브랜치에 merge
git checkout my-setup-binance
git merge my-setup
git push origin my-setup-binance

git checkout my-setup-okx
git merge my-setup
git push origin my-setup-okx

git checkout my-setup-bybit
git merge my-setup
git push origin my-setup-bybit

# 4. 각 폴더에서 pull + 재시작
cd ..\nfi-binance && git pull && docker compose restart
cd ..\nfi-bybit && git pull && docker compose restart
# nfi-okx는 이미 최신
```

## 공식 NFI 업데이트 수용 흐름

```powershell
# 1. main 동기화
cd <아무 폴더>
git fetch upstream
git checkout main
git merge upstream/main
git push origin main

# 2. my-setup에 반영
git checkout my-setup
git merge main
git push origin my-setup

# 3. 각 거래소 브랜치에 반영 (위 "공통 설정 업데이트" 절차)
```

## 운영 상태 확인

```powershell
# 모든 컨테이너 상태
docker ps

# 특정 봇 로그
cd nfi-okx
docker compose logs -f

# 특정 봇 재시작
docker compose restart

# 전부 정지
cd nfi-binance && docker compose down
cd ..\nfi-okx && docker compose down
cd ..\nfi-bybit && docker compose down
```

## 주의사항

### 포트 충돌 방지
- 각 폴더의 `.env`에서 `LISTEN_PORT`를 반드시 다르게 설정
- 기본값: Binance 9044, OKX 9046, Bybit 9045

### BOT_NAME 구분
- sqlite 파일명은 `{BOT_NAME}_{EXCHANGE}_{MODE}-tradesv3.sqlite` 형식
- BOT_NAME 겹치면 동일 파일 공유 시도 → 충돌
- 반드시 거래소별로 다르게: `NFI_DryRun_Binance`, `NFI_DryRun_OKX`, `NFI_DryRun_Bybit`

### 텔레그램 봇 별도 발급
- 거래소별로 각각의 봇 생성 (@BotFather)
- 같은 봇 쓰면 메시지 섞임

### OneDrive 주의
- 현재 구조는 OneDrive 안에 위치
- sqlite 파일 동기화 부하 있을 수 있음
- 문제 생기면 OneDrive 밖으로 이동 고려

## VSCode 권장 열기 방식

한 번에 하나만 작업할 때: 특정 폴더만 열기

File → Open Folder → nfi-okx (또는 해당 폴더)

```
File → Open Folder → nostalgia-for-infinity (부모)
```
- 한 창에서 3개 폴더 모두 보임
- 각 폴더의 .git이 독립적으로 관리됨
